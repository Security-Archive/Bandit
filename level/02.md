### `-`로 시작하는 파일명을 읽지 못하는 이유
리눅스 CLI에서는 `-`로 시작하는 인자를 **옵션**으로 해석한다. 따라서 파일명이 `-`로 시작하면 `cat`은 이를 파일이 아닌 옵션으로 인식한다.

### 해결 방법
1. `./`를 붙여 **경로임을 명시**
```sh
cat ./-파일명
```

2. 입력 리다이렉션(`<`)을 사용해 stdin으로 전달
```sh
cat < -filename
```

3. `--`로 옵션 해석 종료
```sh
cat -- -filename
```
